Jenkins shared library:
=======================
 > A Jenkins Shared Library is a collection of reusable pipeline code (Groovy scripts) stored in a separate Git repository (or inside Jenkins).

 > Instead of duplicating the same pipeline steps in every Jenkinsfile, you write them once in the library and call them whenever needed.

 > Think of it like a “common utilities toolkit” for Jenkins pipelines.

Why Do We Need Shared Libraries?
 > Reusability: One place to write a build function, then use it in 50+ pipelines.
 > Consistency: Every team follows the same process (security checks, build standards, notifications).
 > Maintainability: Update once in the library → applies to all jobs.
 > Scalability: Large companies with hundreds of Jenkinsfiles don’t want every project team to reinvent the wheel.
