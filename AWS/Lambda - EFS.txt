LAMBDA:

In AWS, we have certain primary tools like: EC2, S3, IAM, ELB, Route53 etc., All this tools helps us to create an infrastructure.

On the other hand, we do have some tools like: Code deploy, Code pipeline, Lambda, Cloud formation etc., all this tools are like enhancement/optional tools which is like additional tool that helps to develop our environment acording to the need. (This tools are mostly used by developers, hence coding should be added in order to execute this tools)

LAMBDA = Serverless [It is a place to run our code] on the other words, this will be fully managed by AWS.
In EC2, we are going to do everything manually, when to start/stop it.

Today, we are going to see a static website with login credentials, example: https://www.w3schools.com/
If we are bringing a login page into a website, which means we definetely have Database within it; because that is the space where the user credentails got saved.

In real-time, we have options to upload images into our website, mostly all the applications will have this features, and if we are trying to upload our image with full-size it is going to consume more space into our database as there is a lot of customers; hence we have an option called flatten the image. [example: In whatsapp, we used to share our photo (real size: maybe 2- 3 mb), but on the other  end they will receive the content only in (kb) size]

In real-time, if we are going to give our code inside our EC2 machine, this is going to run for 24*7, because this is a running server; But we are going to replace this with Lambda, this is going to keep our code and when a user is uploading an image, then this code is going to get triggered automatically. Through this we can minimize our cost, as this is going to get charged only for the certain milli-seconds till the code was running.

[https://us-east-2.console.aws.amazon.com/lambda/home?region=us-east-2#/begin]

We got 1million request free in lambda or 4,00,000 GB-seconds of compute per month are free.

According to the number of request, Lambda will automatically increase it's configuration.

EFS - Elastic File system
This is working under the same S3 storage class
This works only in the Linux Env
We are using this to share the volume for multiple instances.
This is working under NFS (Network File Sharing) protocol; where I am going to create a volume under EFS and going to attach to all the instanes. [If a file got created under this volume, it is visible under all the instances; If they are going to delete the file, it is going to get deleted in all the machines], this is called shared volume.
Upto 25,000 files can be created for free under this shared volume.

Steps:
1. Create 2 instances - attach default SG (make sure adding NFS type, if we are using the customized one) - attach key pair - launch instances
2. Create File system - customize - give a name (myefs) - disable back-up - lifecycle management (7days since last access) - performance mode [general purpose: adjust with normal speed; max i/o: provides with full speed access] - throughput mode [bursting: adjust according to the files and network; provisioned: customizing our transfer speed according to our requirement] - disable the encryption - next - under mount targets, delete the default security group and attach to the SG which we created for the server (myefs) on all the AZ's - skip the policy - review and create.
3. Now, we are going to connect to the server, select the instance - click on connect - ec2 instance connect - connect (this will directly connect us to the terminal) [***this option will work only for Amazon Linux, and not for any other OS] - Open the 2nd instance via same pattern.
4. Switch to root user on both the instances (sudo su - root)
5. Create 2 different directory on both the machines seperately (server1: mkdir test) (server2: mkdir new)
6. Now we are going to attach both the servers - go to efs - select the file system - click on attach - (copy the command - using nfs client; paste on both the servers  by changing only the directory name on the end) [example: sudo mount -t nfs4 -o nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2,noresvport fs-be75a5c5.efs.us-east-2.amazonaws.com:/ test] [sudo mount -t nfs4 -o nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2,noresvport fs-be75a5c5.efs.us-east-2.amazonaws.com:/ new]
7. Confirm this by this (df -h) file system command [where you can see under each directory the efs path has been connected]
8. Go inside the directory on both the servers (cd test) (cd new)
9. Confirm on both the server that there is no file present at this moment by giving (ls -ltr)
10. Now, create a new file under 1 machine (touch hi), check the same availability on both the machine, it will be presented on both the machines now.
11. Try the same on another machine (echo "hello world" > akshiv.txt) - check using (ls -ltr) - switch to the next machine and check for the same file using (cat akshiv.txt)
