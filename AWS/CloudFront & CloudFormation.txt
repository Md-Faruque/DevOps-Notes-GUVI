CF - CloudFront (CDN = Content Delivery Network)
	This service is an optional one, depends on company and scenario this may/may not be needed. 

Each and every services will have some catchy words (ex: ec2 = instances, sns = topic, s3 = buckets,objects etc., same like that for CF also we have something called cache servers, edge locations, distributed networks)
what is cache? - this means temporary storage / content.

What is CDN?
	When I am having an content on my S3 bucket in London region, and users from worldwide are going to access that content means this may delay my response time (Latency = response time); that leads to affect my performance, customer satisfaction and business as well. Hence, in order to avoid that we are using a tool called Edge Location.

To know the Edge Locations: https://aws.amazon.com/about-aws/global-infrastructure/
click on the map, it will provide with the entire information about the location details.

What is Edge Location?
	We have data centers in multiples regions all around the world but this edge location means a place where we have additonal servers for better and faster communications.
What is the purpose of Edge Locations?
	This locations helps us to save our data's like cache; by enabling the edge loctions, we don't want to connect to our main storage/server all the time, instead the information's will be saved in the edge location places and via that we will be able to reach our content sooner.

Totally there are 121 edge locations across the globe.

Once, I enabled the edge locations, automatically this places are going to store our content, and through this we can avoid connecting to our origin server everytime.

For example: Go to google and search for (s3 transfer acceleration comparison),  when we check the upload speed normally and with the upload speed on accelearator, you can see a difference on the speed when it works from edge location.
How this works:
When I am trying to upload a file from my location (chennai) to the S3 data center storage location, it will provide us an output; and when we are trying via S3 accelearated transfer then this will be comparitively faster (*** this depends upon our internet speed also) - it's just like a speedtest

By default, AWS has provided us with a seperate connection between the edge location and the origin, when our connection happened via this way, it will be faster by default.

This is a chargable service, (for 1st 12hours it will be free, after that it is going to cost us)

When we are accessing our data for the 1st time, this will reach to the edge location; as there is no information available from there it will reach the origin server, and from the next time we don't want to go to the origin for the same information as it got already got a copy saved under edge location.
[FYI: as I said earlier, this will be a temporary memory, so this is going to be saved there only for sometime and this depends on the TTL (Time To Live), whch we are going to set for the connection]

Interview Question:
1. What if I deleted my origin? will I be able to access my content?
	We can access our data from our edge location, if we are trying to reach the content within that TTL.

Mostly for static data only we are going to use this edge locations; [static data means, the website content are same]; if incase my application are like instagram/facebook [dynamic data], then everytime the content is going to get updated and there is no use of edge location as it is always going to reach the origin server for most of the needs, at that time they will create a seperate servers on the specific locations itself.

A stack is a collection of AWS resources that you can manage as a single unit. In other words, you can create, update, or delete a collection of resources by creating, updating, or deleting stacks. All the resources in a stack are defined by the stack's AWS CloudFormation template.
