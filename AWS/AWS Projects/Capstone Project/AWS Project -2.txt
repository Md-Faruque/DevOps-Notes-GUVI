25. We are going to sync in-between prod and dr and this should not be a manual process, because whenever our prod is going to go down dr need to pick up the data, this will happen only they are in sync, hence for doing this we are going to create crontab (cron means doing activities automatically) - open prod server - go to (cd wp-content) [this is the place where our app are actually running] - go inside (cd uploads) [this is the place where we are going to save our codes] - give command (crontab -l) [this will show currently there is no crontab for this app] - now we are going to create one, give command (crontab -e) [this command means, we are going to edit the cronjob] - click (i) [insert] - paste this job 
[*/2 * * * * aws s3 sync --delete /var/www/html/wp-content/uploads s3://wordpress-code-asquaretech
*/2 * * * * aws s3 sync --delete /var/www/html/ s3://wordpress-media-asqauretech]
give command (crontab -l) [now you can see the job]

26. Now, go to the dr machine and set up the sync connection over there also, same method, go to (cd wp-content) [this is the place where our app are actually running] - go inside (cd uploads) [this is the place where we are going to save our codes] - give command (crontab -l) [this will show currently there is no crontab for this app] - now we are going to create one, give command (crontab -e) [this command means, we are going to edit the cronjob] - click (i) [insert] - paste this job 
[*/2 * * * *  aws s3 sync --delete  s3://wordpress-code-asquaretech/var/www/html/wp-content/uploads
*/2 * * * *  aws s3 sync --delete  s3://wordpress-media-asqauretech/var/www/html/]
give command (crontab -l) [now you can see the job]
