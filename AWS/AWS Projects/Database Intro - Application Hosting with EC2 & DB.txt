Database Fundamentals:

What is Data?
	Data can be anything or everything, any information or fact can be considered as data, (ex: name, age, DOB etc.,) and these can be stored into Database. Data can be any form like image, plain text, video, file. Now we need to know where to store this data and the purpose of using it.

What is Database?
	Database can be considered as a container filled with data or information which is electronically stored inside a computer, THe purpose of storing these data is to easily access, modified, protected and alaysed. In our day-to-day basis we are using/accessing multiple databases, because most of softwares has become mobile applications.

Is Database same as DBMS?
	Database is just a container to store data, whereas DBMS (Database Management System) is a software used to manage our database, we need DBMS to communicate with our database in order to store, modify, protect & retrieve your data's. ex: MySQL, PostgreSQL, MongoDB, Neo4j, Cassandra etc., every DBMS has their own set of language to communicate with your data's. 

Evolution of Database: (1960s)
Flat File Database: data's were stored in simple structure like .csv, text etc.,
Hierarchical Database
Network Database: Both these databases were incapable of storing complex data relationships. hence soon it was replaced by Relational Database. 

Types of Database:
1. Relational === Over 70% of data used today belongs to this category, 
2. Non-Relational == Over the huge incerase in data recently this Non-relational database becomes popular, (mainly because of social media platforms); 

Companies are even combining both these database according to their requirement. 
Oracle is the most widely used Relational Database
MongoDB is the highly used Non-Relational Database

Relational Database, data's are stored through collection of tables, each informational theme (business concept) is stored in its own table, these tables are related to one another. Each table consist of a Columns & Rows. Each Column has a Name and Data Type (is a data rule which is associated to every column; only those data's satisfy this rule can be inserted in the specific column. Row can be treated as a record which is formed by single or multiple columns. 

Non-Relational Databse, there are variety of range of this, such as Key value Database, Document Database, Graph Database, Wide Column Database, Search Engine Database, Time Series Database etc., each of these databases stores differently and are useful to manage data's specifically.  

	Key Value store == simplest database, where every data will be assigned to a key. These are useful to store certain type of data such as, configuration data, state information, any data represented by a dictionary or a # "hash" in any programming language. (ex: Redis, Memcached)
	
	Document store == this also use unique key to identify database, however unlike key-value store this database store, store their data in structured format store called documents often using .Json, XML etc., (ex: Mongo DB, COuchDB etc.,)

	Graph Database == this brings relations by following nodes, edges & properties. (ex: Neo4j database)

	Wide Column Database (or) Column family database: this use rows and columns but they don't use tables here. Instead of tables they use structure called as column families (ex: Cassandra, HBase etc.,)

SQL Basics:
	This is the core of Relational database properties, stands for Structured Query Language; this was developed by IBM in the early 1970s. THis is powerful language that uses simple english sentences, this is a Declarative language. They maintain well-defined standards.

<?php

/* Add an employee to the table. */
function AddEmployee($connection, $name, $address) {
   $n = mysqli_real_escape_string($connection, $name);
   $a = mysqli_real_escape_string($connection, $address);

   $query = "INSERT INTO EMPLOYEES (NAME, ADDRESS) VALUES ('$n', '$a');";

   if(!mysqli_query($connection, $query)) echo("<p>Error adding employee data.</p>");
}

/* Check whether the table exists and, if not, create it. */
function VerifyEmployeesTable($connection, $dbName) {
  if(!TableExists("EMPLOYEES", $connection, $dbName))
  {
     $query = "CREATE TABLE EMPLOYEES (
         ID int(11) UNSIGNED AUTO_INCREMENT PRIMARY KEY,
         NAME VARCHAR(45),
         ADDRESS VARCHAR(90)
       )";

     if(!mysqli_query($connection, $query)) echo("<p>Error creating table.</p>");
  }
}

/* Check for the existence of a table. */
function TableExists($tableName, $connection, $dbName) {
  $t = mysqli_real_escape_string($connection, $tableName);
  $d = mysqli_real_escape_string($connection, $dbName);

  $checktable = mysqli_query($connection,
      "SELECT TABLE_NAME FROM information_schema.TABLES WHERE TABLE_NAME = '$t' AND TABLE_SCHEMA = '$d'");

  if(mysqli_num_rows($checktable) > 0) return true;

  return false;
}
